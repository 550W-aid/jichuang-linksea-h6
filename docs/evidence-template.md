# 测试证据与答辩素材模板

这份模板用于把 bring-up、算法效果和最终答辩素材统一收口到 `evidence/`。

## 建议目录

```text
evidence/
|-- board-bringup/   板卡照片、下载成功截图、串口回读
|-- camera/          摄像头画面、颜色/方向/同步验证
|-- demo/            原图、处理后图、参数配置截图
|-- metrics/         资源利用率、时序、FPS、寄存器快照
`-- video/           演示视频、剪辑片段、旁白稿
```

## 文件命名规则

统一使用：

```text
YYYYMMDD-阶段-场景-模式-说明.ext
```

例如：

- `20260326-bringup-vga-testpattern-monitor.jpg`
- `20260327-camera-desk-bypass-raw.jpg`
- `20260327-demo-darkroom-lowlight-enhanced.jpg`
- `20260327-metrics-lowlight-fps-registers.txt`

## 每个阶段至少保留的证据

| 阶段 | 最少保留内容 |
|---|---|
| 板级 bring-up | 板卡上电照片、下载成功截图、UART 回读、VGA 测试图 |
| 摄像头链路 | 原始画面、颜色/方向验证图、异常现象记录 |
| 算法演示 | 原图、处理后图、寄存器配置、FPS |
| 指标整理 | 资源利用率、时序报告、30 分钟稳定性结果 |
| 最终答辩 | 板卡接线图、系统框图、演示视频、亮点页截图 |

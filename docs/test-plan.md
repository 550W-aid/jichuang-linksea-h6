# 测试计划

## 1. 板级测试

- 下载 `link_sea_h6_bringup_top`，确认 LED 心跳正常。
- 通过串口执行 `ping`、`read`、`write`，确认寄存器回读一致。
- VGA 输出彩条或测试画面，确认显示器锁定 `640x480@60`。
- 将下载截图、串口回读和 VGA 照片保存到 `evidence/board-bringup/`。

## 2. 摄像头链路测试

- 接入 `OV5640` 后检查初始化完成标志。
- 验证采集图像方向、颜色空间和场同步是否正确。
- 连续运行 30 分钟，观察是否出现花屏、拖影、黑屏。
- 将原始画面和异常现象记录保存到 `evidence/camera/`。

## 3. 算法测试

每个算法都至少保留以下证据：

- `原图`
- `处理后图`
- `寄存器配置`
- `帧率`

建议统一归档到 `evidence/demo/` 与 `evidence/metrics/`。

建议至少拍摄以下场景：

- 正常光照室内
- 暗光走廊或桌面
- 有明显边缘的物体
- 需要放大观察的局部细节

## 4. 性能测试

记录以下指标：

- 综合后资源利用率
- 最差时序裕量
- 直通模式帧率
- 低照增强模式帧率
- 边缘叠加模式帧率

建议把报告截图和寄存器快照统一保存到 `evidence/metrics/`。

## 5. 答辩演示顺序

1. 板卡、摄像头、显示器实物展示
2. 直通画面
3. 基础算法轮播
4. 低照增强前后对比
5. 巡检叠加模式
6. 串口或 STM32 实时调参
7. 指标页和资源占用页
